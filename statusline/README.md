# Statusline

A portable `statusline.sh` that Claude Code runs every second or so to render the bottom status bar. Shows: current directory, git branch, model, context usage, token burn rate, session cost.

## Contents

- [`statusline.sh`](statusline.sh) — the script; portable, no project specifics.

## Install

```bash
cp statusline/statusline.sh ~/myproject/.claude/statusline.sh
chmod +x ~/myproject/.claude/statusline.sh
```

Wire it in the project's `.claude/settings.json`:

```json
{ "statusLine": { "type": "command", "command": ".claude/statusline.sh" } }
```

## What it displays

```
 my-project  main  Opus 4.7  32k/200k  $0.42/h
```

| Field | Source |
|---|---|
| Directory | `basename "$CLAUDE_PROJECT_DIR"` |
| Git branch | `git branch --show-current` |
| Model | `$CLAUDE_MODEL_DISPLAY_NAME` |
| Context usage | `$CLAUDE_INPUT_TOKENS / $CLAUDE_MAX_CONTEXT_TOKENS` |
| Cost burn rate | derived from `$CLAUDE_COST_USD` and session duration |

## Dependencies

- `bash` (any recent version)
- `jq` (optional; degrades gracefully if missing)
- `git` (optional; shows `-` if not a git repo)
- A terminal supporting 256-color ANSI codes

## Customizing

Environment variables at the top of `statusline.sh` control colors and field visibility:

```bash
SL_COLOR_DIR=117    # 256-color code for the directory segment
SL_COLOR_BRANCH=178
SL_SHOW_COST=1      # set to 0 to hide cost
SL_SHOW_TOKENS=1
```

Forks of the script are encouraged — the original is generated by `cc-statusline` and customized. If your fork works well, send it back.
